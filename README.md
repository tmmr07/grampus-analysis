# 本プロジェクトの説明
本プロジェクトは，私が応援している「名古屋グランパス」の各種データを分析するためのデスクトップアプリケーションを開発するプロジェクトです．

# 開発目的
本アプリの開発は，以下の3点を主な目的としています．

- Pythonの習得: Pythonを用いた実践的なデスクトップアプリ開発スキルの向上
- 統計学の学習: データ分析を通じた統計手法の理解と応用
- アジャイル開発の実践: 近年の主流である開発プロセス（アジャイル開発）の体験と学習

# 開発手法について
本プロジェクトでは，開発目的にも挙げている通り，近年ソフトウェア開発の現場で急速に普及しているアジャイル開発という手法を採用して進めていきます．

アプリケーション開発の代表的なアプローチとして，従来からよく知られている「ウォーターフォール開発」と，今回採用する「アジャイル開発」があります．それぞれの特徴は以下の通りです．

- **ウォーターフォール開発とは**
要件定義，設計，実装，テストといった各工程を，上から下へ滝（ウォーターフォール）のように順番に完了させていく伝統的な手法です．開発の初期段階で全体の計画や仕様を細かく決定するため，全体像やスケジュールを把握しやすいというメリットがあります．その一方で，開発の途中で仕様変更が生じた場合，前の行程に戻って修正を行うことが難しく，変更コストが大きくなりやすいという側面を持っています．

- **アジャイル開発とは**
小さな機能単位で「計画・設計・実装・テスト」という短い開発サイクルを反復（イテレーション）しながら，少しずつソフトウェアを完成させていく手法です．開発途中の要件変更や追加に対して柔軟かつ迅速に対応できるのが最大の強みです．常に動作するソフトウェアを確認しながら改善を重ねることができるため，より実用的で価値のあるプロダクトを作り上げることができます．

本プロジェクトでは，このアジャイル開発の「柔軟に変更を受け入れ，改善を繰り返す」というプロセスを実際に体験しながら，データ分析アプリを完成させていくことを目指します．
